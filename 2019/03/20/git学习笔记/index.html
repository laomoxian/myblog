git(分布式版本控制系统) vs CVS/SVN(集中式版本控制系统)

集中式版本控制系统：版本库集中存放于中央服务器中，在工作时，需要先从服务器获取到最新的版本，在工作完毕后把的内容更新到服务器中。必须在联网的条件下才能工作。

分布式版本控制系统：没有中央服务器，没人的电脑都有完整的版本库，不需要在联网的环境下工作，版本的控制通过相互的推送修改实现。

所有的版本控制系统都只能跟踪文本文件的改动，而不能跟踪二进制文件的变化。其中，word为二进制文件。 避免使用 windows的记事本来编辑任何文本文件。

git 的安装

在Linux上安装Git
1、使用 git 查看是否已安装git
2、sudo apt-get install git （适用Debian 、Ubuntu）

在windows上安装Git
1、从官网下载：https://git-scm.com/downloads

2、安装完成后在 bit bash 或命令行输入
$ git config –global user.name “Your Name”
$ git config –global user.email “email@example.com
（其中使用git config命令的–global参数，表示你这台机器上所有的Git仓库都会使用这个配置）

一、创建版本库
版本库：又名仓库（repository），可以理解为由git管理起来的一个目录，可以跟踪每个文件的修改、删除，以便追踪历史或还原。

1、在合适的地方创建一个目录：
如：
$ mkdir learngit //创建文件夹
$ cd learngit //进入文件夹
$ pwd //显示路径

ps:windows系统下为了避免遇到一些问题，确保目录不包含中文。

2、$ git init
把该目录变成Git可以管理的仓库
(创建成功后，目录会出现.git目录)

3、将文件添加到文本库。
例子：
编写readme.txt
$ git add readme.txt
$ git commit -m “wrote a readme file”
git commit命令中，-m后面输入的是本次提交的说明。
ps:可以一次提交多个文件。

命令：
git status 查看仓库的当前状态，告诉你有文件被修改过
git diff <文件名> 查看文件的修改内容
git reset –hard commit_id 版本间的穿梭，其中HEAD指向的版本为当前版本
git log 查看提交历史
git reflog 查看命令历史
git rm 删除一个文件

修改撤销：
场景1：当你改乱了工作区某个文件的内容，想直接丢弃工作区的修改时，用命令git checkout – file。

场景2：当你不但改乱了工作区某个文件的内容，还添加到了暂存区时，想丢弃修改，分两步，第一步用命令git reset HEAD ，就回到了场景1，第二步按场景1操作。

场景3：已经提交了不合适的修改到版本库时，想要撤销本次提交，参考版本回退一节，不过前提是没有推送到远程库。

分支：

查看分支：git branch

创建分支：git branch

切换分支：git checkout

创建+切换分支：git checkout -b

合并某分支到当前分支：git merge

删除分支：git branch -d

当Git无法自动合并分支时，就必须首先解决冲突。解决冲突后，再提交，合并完成。

解决冲突就是把Git合并失败的文件手动编辑为我们希望的内容，再提交。

用git log –graph命令可以看到分支合并图。

合并分支时，加上–no-ff参数就可以用普通模式合并，合并后的历史有分支，能看出来曾经做过合并，而fast forward合并就看不出来曾经做过合并

修复bug时，我们会通过创建新的bug分支进行修复，然后合并，最后删除；

当手头工作没有完成时，先把工作现场git stash一下，然后去修复bug，修复后，再git stash pop，回到工作现场。

开发一个新feature，最好新建一个分支；

如果要丢弃一个没有被合并过的分支，可以通过git branch -D 强行删除。
