  • 软件测试的概念和分类

    软件需经过严密的测试，以发现软件在整个设计过程中存在的问题并加以纠正。

  • 测试用例的概念、分类、不同测试阶段采用哪些类型的测试用例

    概念：

    测试用例是测试执行的最小实体，是为特定的目的而设计的一组测试输入、执行条件和预期的结果。
    其目的是确定应用程序的某个特性是否正常的工作，并且达到程序所设计的结果。

    分类：

    测试用例归为5大类：

    1. 白盒测试用例：逻辑覆盖法、基本路径法
    2. 软件各项功能的测试用例：黑盒
    3. 用户界面测试用例
    4. 软件的各项非功能测试用例
    5. 对软件缺陷修正所确认的测试用例
    不同测试阶段

  • 软件缺陷概念。复杂软件缺陷的生命周期

    软件缺陷：
    软件缺陷简单说就是存在于软件（文档、数据、程序）之中的那些不希望，或不可接受的偏差，而导致软件产生的质量问题。

    软件缺陷生命周期：
    软件缺陷的生命周期指的是一个软件缺陷被发现，报告到这个缺陷被修复、验证直到最后关闭的完整过程。

  • 为什么不是所有软件缺陷发现就能修复？

    1.没时间
    2.不是真正的软件缺陷
    3.修复风险大
    4.不值得修复
